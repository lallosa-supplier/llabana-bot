1. RESUMEN DEL PROYECTO
═══════════════════════════════════════════════════════════════════════════════

Chatbot de WhatsApp para Comercializadora Llabana (empresa mexicana de alimento
balanceado). Los clientes escriben por WhatsApp (Twilio), el bot usa Claude AI
con tool calling para entender qué necesitan, consulta Google Sheets para datos
de clientes y productos, maneja escalaciones a asesores humanos (Wig), e integra
órdenes de Shopify. Corre en Node.js/Express, desplegado en Railway.

3. FLUJO DE UN MENSAJE (paso a paso)
═══════════════════════════════════════════════════════════════════════════════

ENTRADA:
────────
1. Cliente escribe en WhatsApp: "Hola, quiero info"

TWILIO → WEBHOOK:
──────────────────
2. Twilio envía POST a /webhook/whatsapp (webhookHandler.js)
3. webhookHandler extrae: from=+521234567890, body="Hola, quiero info"
4. Deduplica por MessageSid (últimos 100 únicos)
5. Responde a Twilio con 200 inmediato (async, no bloquea)
6. Inicia debounce 1.5s (acumula mensajes rápidos)

PROCESAMIENTO:
───────────────
7. webhookHandler.procesarMensaje() se ejecuta
8. Carga transcripción previa de "5 Transcripciones"
9. Carga o crea sesión: sessionManager.getSession(phone)

MANEJO DE MENSAJE:
───────────────────
10. botLogic.handleMessage(phone, "Hola, quiero info")
    → DELEGA a aiBot.handleMessageIA()

IA CON TOOL CALLING:
─────────────────────
11. aiBot.handleMessageIA(phone, "Hola, quiero info"):
    - Obtiene sesión actual
    - Agrega mensaje a history: { role: 'user', content: "Hola..." }
    - Construye MASTER_PROMPT + catálogo + FAQs
    - Envía a Claude Sonnet con 3 herramientas

12. Claude analiza:
    - Entiende que quiere información
    - Detecta que no tiene nombre/CP aún
    - Responde: "¿Con quién tengo el gusto?"
    - NO usa herramientas (stop_reason != 'tool_use')

13. Retorna respuesta: "¿Con quién tengo el gusto?"

PERSISTENCIA:
──────────────
14. aiBot guarda en history: { role: 'assistant', content: "¿Con quién...?" }
15. sessionManager.updateSession() guarda sesión con nueva history
16. transcriptService.updateTranscript() guarda en "5 Transcripciones"

RESPUESTA AL CLIENTE:
──────────────────────
17. webhookHandler envía por Twilio:
    twilioService.sendMessage(phone, "¿Con quién tengo el gusto?")
18. Cliente ve respuesta en WhatsApp

SIGUIENTE MENSAJE:
───────────────────
19. Cliente responde: "Juan García"
20. Se repite desde paso 7, pero ahora history tiene contexto previo
21. Claude detecta nombre, llama registrar_o_actualizar_cliente (herramienta 2)
22. Herramienta ejecuta sheetsService.registerCustomer()
23. Cliente se crea en "1 Base Maestra"
24. Claude pregunta por CP


ESCALACIÓN (si aplica):
────────────────────────
Si cliente dice "quiero entrega directa" o "urgente":
25. Claude llama herramienta 3: escalar_a_wig(motivo, resumen)
26. botLogic.notifyWig() se ejecuta:
    - Verifica horario con horarioService.estaEnHorario()
    - Si fuera de horario: colaEscalaciones.agregarEscalacion()
    - Si en horario: twilioService.sendMessage(WIG_WHATSAPP_NUMBER, mensaje)
27. Wig recibe alerta en su WhatsApp
28. sessionManager.updateSession() marca flowState = 'waiting_for_wig'
29. Cliente recibe: "Un asesor te contactará en breve"

MASTER_PROMPT (instrucciones de Claude):
─────────────────────────────────────────

El prompt le dice a Claude QUÉ es (asesor de Llabana) y CÓMO debe actuar:

1. TONO:
   - Directo y sencillo (platicando con gente de campo)
   - Usa "tú", frases cortas
   - Emojis con medida
   - Entiende errores ortográficos

2. FLUJO DE CONVERSACIÓN:
   Paso 1: Saludar + capturar NOMBRE + APELLIDO + CP
     - "Juan" sólo no vale → pide apellido
     - Palabras como "gato", "perro", "gallina" NO son nombres
     - En cuanto tiene nombre+apellido → llama registrar_o_actualizar_cliente

   Paso 2: Llamar consultar_zona(cp) SIEMPRE que tenga CP
     Según resultado:

   a) "entrega_directa" (CDMX, Ecatepec):
      → Recoge qué producto y cuánto necesita
      → Llama escalar_a_wig()
      → Cliente ve: "Un asesor lo contactará"

   b) "paqueteria" (resto de México):
      → Distingue 2 perfiles:

         CLIENTE FINAL (mascota, <10 bultos):
         → LO ATIENDE SOLO
         → Recomienda 1 opción clara del catálogo
         → Calcula bultos necesarios
         → Da precio real
         → Manda link llabanaenlinea.com
         → Registra segmento "Cliente final"

         MAYOREO/NEGOCIO (quiere revender, descuentos, >10 bultos):
         → Dice HONESTO: "Por ahora solo hacemos pedidos chicos"
         → NO manda link
         → Registra segmento "Mayoreo fuera de zona"
         → Cierra con cortesía

3. REGLAS CRÍTICAS:
   - Nunca revelar: cuántas sucursales, en qué estados está
   - Nunca prometer: día exacto, entrega garantizada
   - Nunca inventar: precios (usa catálogo real)
   - Si hay queja/enojo/frustración → escalar a Wig


HERRAMIENTAS (3 funciones que Claude puede llamar):
────────────────────────────────────────────────────

1. consultar_zona(cp)
   - INPUT: cp (código postal 5 dígitos)
   - OUTPUT: JSON { zona: 'entrega_directa'|'paqueteria', estado, ciudad }
   - LÓGICA:
     const ZONA_ENTREGA_CPS = ['550','551','552','555','540','541','06'];
     if (cp.startsWith cualquier CPs) → 'entrega_directa'
     else → 'paqueteria'
   - LLAMADA INTERNA: sheetsService.lookupCpMX(cp)


2. registrar_o_actualizar_cliente(input)
   - INPUT: { nombre, apellido, cp, segmento, notas }
   - LÓGICA:
     const nombreCompleto = [nombre, apellido].join(' ')
     const existing = sheetsService.findCustomer(phone)
     if (existe) → sheetsService.updateOrderData(rowIndex, campos)
     else → sheetsService.registerCustomer({ phone, name, nombreCompleto, ... })
   - OUTPUT: "Cliente registrado." o "Cliente actualizado."


3. escalar_a_wig(input)
   - INPUT: { motivo, resumen }
   - LÓGICA:
     await botLogic.notifyWig(phone, session, motivo, resumen)
     await sessionManager.updateSession(phone, { flowState: 'waiting_for_wig' })
   - OUTPUT: "Escalado a Wig. Un asesor lo atenderá."
   - NOTA: Esto dispara notifyWig() que:
     - Si fuera de horario → guarda en Redis (colaEscalaciones)
     - Si en horario → envía WhatsApp a Wig


LOOP DE TOOL CALLING:
─────────────────────

Pseudocódigo:

for (iteración = 1; iteración <= 5; iteración++) {

  // Envía a Claude CON herramientas
  response = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 1024,
    system: MASTER_PROMPT + catálogo + FAQs,
    tools: [ consultar_zona, registrar_o_actualizar_cliente, escalar_a_wig ],
    messages: history
  })

  // Extrae texto si hay
  textos = response.content
    .filter(b => b.type === 'text')
    .map(b => b.text)
  if (textos.length) finalText = textos.join('\n')

  // Si Claude NO quiere usar herramientas → fin
  if (response.stop_reason !== 'tool_use') break;

  // Claude QUIERE usar herramientas → ejecutarlas
  messages.push({ role: 'assistant', content: response.content })

  results = []
  for (cada tool_use en response.content) {
    salida = ejecutarHerramienta(tool_use.name, tool_use.input, phone, session)
    results.push({ type: 'tool_result', tool_use_id: tool_use.id, content: salida })
  }

  // Responde a Claude con resultados
  messages.push({ role: 'user', content: results })
}

// Fin: Claude respondió sin herramientas
history.push({ role: 'assistant', content: finalText })
sessionManager.updateSession(phone, { conversationHistory: history })
return finalText
